Qué es Pillars ?

PillarsJS es un Framework de Desarrollo Web, modular y escalable, en Node.JS.

Contruye verdaderas aplicaciones Web MVC.

Lee el alcance de Pillars, instala y comienza con los primeros ejemplos.

Metodología

En Pillars hay cuatro entidades básicas:

  • App (Aplicación): se definen parámetros globales de la aplicación como puerto al que responde, base de datos o idiomas de la aplicación.
  • Pillar (Pilar): conjunto lógico de rutas.
  • Beam (Viga): Un Beam (viga) siempre se contruye sobre un Pillar, y la definimos como un conjunto de acciones que se ejecutan sobre la ruta del Pillar.
  • Gangway (escalerilla): es una abstracción del conjunto resquest/response propio de node.js.

Debes tener Node.JS y MongoDB instalados previamente. Puedes encontrar instrucciones de instalación aquí.

Configuraciones de Pillars

Pillars permite una serie de configuraciones:

  • uploadsDirectory: Directorio del servidor que se utilizará para la subida de ficheros. Por defecto: ./uploads
  • tempDirectory: Directorio del servidor donde se suben los ficheros temporales. Por defecto: ./temp
  • maxUploadSize: Tamaño máximo del archivo a subir al el servidor. Por defecto: 10*1024*1024.
  • maxGzipSize: Tamaño máximo del archivo a comprimir por el servidor al realizar el envío al cliente. Por defecto 5*1024*1024.
  • htmlErrors: true para debug on. Por defecto: true.

 pillars.configure({
 	uploadsDirectory: "./uploads",
 	tempDirectory: "./temp",
 	maxUploadSize: 10*1024*1024,
 	maxGzipSize: 5*1024*1024,
 	htmlErrors : true
 });

Clase AppVer Código

La Clase App define parámetros básicos de nuestra aplicación.

Hola que tal!


 var app = new pillars.App();
 app.start(80,'www.example.com');
 console.log(app.port,app.pillars); //> 80, [pillar1,pillar2]
 app.stop();
Propiedades
  • App.prototype.port: puerto en el que se ejecutará.
  • url: url destino que atenderá.
  • app.pillars; array de objetos organizados por prioridad, no en el orden que se añadieron, con los pilares que tiene esa aplicación.
Métodos
  • App.prototype.start(): Arrancar el servidor. Con una misma app, si se realiza start() sobre puertos diferentes, la segunda llamada realizará un stop sobre la app en el primer puerto.
  • stop: Parar el servidor.
  • add: Añadir un Pillar.
  • languages: Definición de los idiomas en los que funcionará el sistema.
  • database: Base de datos que se lincará a la aplicación.
Curiosidades
  • El servidor HTTP, al realizar el envío de datos al cliente, comprime tanto datos como archivos a enviar. Si no se le pueden enviar los datos comprimidos al cliente, los envía sin comprimir, en caso de poder, los envía comprimidos deflate o g-zip, con prioridad de deflate. En el caso de los archivos, la versión comprimida de éstos se guardan en el disco del servidor para no tenerlo que volver a comprimir.
Ejemplo de App

 var app = new pillars.App(3001,'www.pillarsjs.com');
 app.languages = ['es','en'];
 app.database = 'databasename';	//¿Debe estar creada previamente la base de datos? - Necesario tener mongodb instalado en el servidor.
 app.start();
 app.stop();
Ejemplo 2. Arracar 2 servidores.

 var pillars = require('pillars').global();

 var app = new pillars.App();

 app.add(new Pillar().add(new Beam(function(){
	gw.send("Hola Mundo");
 })));

 app.start(3000); 
 app.start(3001); // Realiza un app.stop() antes del start(3001);

La forma correcta sería instanciando dos aplicaciones.


 var pillars = require('pillars').global();

 var app = new pillars.App();
 var app2 = new pillars.App();

 app.add(new Pillar().add(new Beam(function(){
	gw.send("Hola Mundo");
 })));

 app.start(3000); 
 app2.start(3001);

Con el código anterior tenemos dos aplicaciones ejecutando el mismo código respondiendo en puertos diferentes.

Pillar: Uso, Propiedades y MétodosVer Código

Objetivo

Un Pillar es una clase de organización lógica de rutas.

Uso

 var myPillar = new Pillar();
Propiedades de un Pillar

beams. ejemplo pillar.beams; devuelve una lista ordenada por prioridad de todos los beams que tiene declarado el pilar.


 var myPillar = new Pillar({
 	id: 	'string: idname para el Pillar',
 	path: 	'string: path al que responderá el Pillar. 	Default: / ',
 	host:	'string: host para el que se ejecutará. 	Default: *'
 });
Métodos de un Pillar

Los métodos son add y remove, para añadir y eliminar Beam. Ver ejemplo 1 de Beam.

Ejemplo 1

Creación de un Pillar que responde a la ruta /noticias/


 var myPillar = new Pillar({
 	id: 	'noticias',
 	path: 	'/noticias/'
 });
Ejemplo 2

Creación de un Pillar que responde a cualquier ruta


 var myPillar = new Pillar({
 	id: 	'default',
 	path: 	'*'
 });

Beam: Uso, Propiedades y Métodos.Ver Código

Objetivo

Porro número 4, se le ocurre a Bifuer meter Vigas...

Uso

Un Beam siempre se añade a un Pillar.


 //Instancia del Pillar
 var MyPillar = new Pillar();

 //Instancia del Beam
 var myBeam = new Beam();

 //Añadiendo un Beam a un Pillar
 myPillar.add(myBeam);
Propiedades

 var myBeam = new Beam({
 	id:		'string: identificador del Beam.',
 	path:		'string: path del Beam, dentro del path del Pillar al que esté añadido.		Default:'/'',
 	method:		'string: get o post, o ambos.							Default: get',
 	session:	'boolean: true si levanta el sistema de sesiones, false en caso contrario.	Default: false.',
 	upload:		'boolean: true si permite upload de ficheros.					Default: false.',
 	maxlength:	'integer: tamaño máximo del fichero a subir.					Default: 50*1024*1024 (50MB)',
 	account:	'boolean: true si es necesario que el usuario esté-' 
 			'-logueado para acceder al Beam. Levanta el sistema de sesiones.		Default: false.'
 });
Métodos

No tiene métodos públicos.

Ejemplo 1

Aplicación Completa que muestra en el navegador: Hola Mundo.


var pillars = require('pillars').global();

var app = new pillars.App();

var myPillar = new Pillar();
var myBeam = new Beam(function(gw){
	gw.send("Hola Mundo");
});

myPillar.add(myBeam);
app.add(myPillar);

app.start();
Reutilización de un Beam

Cómo un mismo Beam se puede enganchar a distintos Pillar. En el siguiente ejemplo tenemos un Beam que escribe "Hola Mundo" en el navegador del cliente. Y dos Pillar, uno que enrouta a /test1/ y otro que enrouta a /test2/. En ambas rutas aparecerá el mensaje "Hola Mundo".


var pillars = require('pillars').global();

var app = new pillars.App();

var myPillar = new Pillar({path:"test1"});
var myPillar2 = new Pillar({path:"test2"});

var myBeam = new Beam(function(gw){
	gw.send("Hola Mundo");
});

myPillar.add(myBeam);
myPillar2.add(myBeam);

app
  .add(myPillar)
  .add(myPillar2);

app.start();

Sitio Web de una sola Página HTML

En este ejemplo, la aplicación enviaría al cliente la página home.html. El cliente ha tenido que hacer la consulta: www.sitioweb.com:3000, ya que por defecto el servidor se arranca en este puerto.

Por otra parte, en este ejemplo, sólo tenemos el archivo HTML, sin css en otro directorio ni librerías adicionales, por lo que si tuviera css debería de estar en la misma página HTML.

En este ejemplo hacemos uso del método file de Gangway.


var pillars = require('pillars');

var Pillar = pillars.Pillar;
var Beam = pillars.Beam;

var app = new pillars.App();

app
 .add(new Pillar()
	.add(new Beam(function(){
	 gw.file("./home.html");
	}))
 );

 app.start();

Un Sencillo Servidor de Ficheros Estáticos

Pillars cuenta con un servidor de ficheros estáticos implementado. Estas funcionalidades ya implementadas las llamamos precasts.

Teniendo el siguiente sistema de directorios, suponemos que quiero listar sólo la carpeta web: 


 + pillars-example3
 ---- + node_modules
 ---- + web
 ---- app.js
 ---- packege.json
 ---- readme.md

El código para que http://localhost:3000 liste la carpeta web es el siguiente:


var pillars = require('pillars').global();

var app = new pillars.App();

app
 .add(new Pillar()
  .add(new Beam({path:'*:path',directory:"./web/"},precasts.directory)
  )
 )

app.start();

El precasts en concreto para tener un servidor de ficheros estáticos es precasts.directory.
